# 📚 Prompt Library Guide

Hướng dẫn khai thác thư viện Prompt được thiết kế riêng cho các dự án công nghệ phổ biến.

---

## 📂 Các nhóm Prompt có sẵn

- **ASP.NET Core ([aspnet.md](../prompts/aspnet.md))**: Tối ưu cho thiết kế dự án Clean Architecture, xử lý logic CQRS qua MediatR, cấu hình Fluent API cho EF Core và xử lý Dependency Injection/Logging.
- **React & Next.js ([react.md](../prompts/react.md))**: Component TypeScript, Next.js App Router, state management (Zustand/Context), Tailwind theming, testing với RTL.
- **Python & Django ([python.md](../prompts/python.md))**: Django REST API, FastAPI endpoints, SQLAlchemy design, pytest testing, package structure, authentication.
- **Go (Golang) ([go.md](../prompts/go.md))**: REST API server, database operations, concurrency patterns, testing/mocking, CLI với cobra, module structure.
- **Flutter ([flutter.md](../prompts/flutter.md))**: Định hình cấu trúc thư mục Feature First, tạo State Notifier bằng Riverpod, tạo Immutable data model qua Freezed và thiết lập GoRouter.
- **Unity ([unity.md](../prompts/unity.md))**: Hướng dẫn viết scripts tuân thủ nguyên lý SOLID, thiết kế ScriptableObjects, tối ưu hóa bộ nhớ qua Object Pooling và Zenject Dependency Injection.
- **SQL Server ([sql.md](../prompts/sql.md))**: Tối ưu hóa các truy vấn SQL chậm, cấu hình chỉ mục thông minh (Index Design), viết SQL Migration an toàn và phân tích Execution Plan.
- **Docker & DevOps ([docker.md](../prompts/docker.md))**: Đóng gói multi-stage Dockerfile siêu nhẹ, viết docker-compose môi trường production và thiết lập Health Checks cho container.
- **Clean Code ([clean-code.md](../prompts/clean-code.md))**: Nguyên tắc viết code sạch, đặt tên, cấu trúc hàm, xử lý lỗi và comment hiệu quả.
- **Code Review ([review.md](../prompts/review.md))**: Checklist review code toàn diện — bảo mật, hiệu năng, kiến trúc, khả năng bảo trì.
- **Testing ([testing.md](../prompts/testing.md))**: Chiến lược testing tổng quát — unit test, integration test, end-to-end, mock/stub patterns.

---

## 🛠️ Cách sử dụng Prompt đạt hiệu quả cao nhất
1. Xác định ngôn ngữ/framework đang làm việc.
2. Mở file tương ứng trong thư mục `prompts/`.
3. Điền các thông tin trong ngoặc vuông `[...]` bằng ngữ cảnh thực tế dự án của bạn.
4. Gửi đoạn prompt hoàn chỉnh cho Codex CLI để nhận được kết quả tốt nhất.

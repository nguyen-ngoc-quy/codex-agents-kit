# 📚 Prompt Library Guide

Guide to leveraging the Prompt Library designed for popular technology stacks.

---

## 📂 Available Prompt Groups

- **ASP.NET Core ([aspnet.md](../../prompts/aspnet.md))**: Clean Architecture design, CQRS with MediatR, Fluent API for EF Core, Dependency Injection/Logging.
- **React & Next.js ([react.md](../../prompts/react.md))**: TypeScript components, Next.js App Router, state management (Zustand/Context), Tailwind theming, RTL testing.
- **Python & Django ([python.md](../../prompts/python.md))**: Django REST API, FastAPI endpoints, SQLAlchemy design, pytest testing, package structure, authentication.
- **Go (Golang) ([go.md](../../prompts/go.md))**: REST API server, database operations, concurrency patterns, testing/mocking, cobra CLI, module structure.
- **Flutter ([flutter.md](../../prompts/flutter.md))**: Feature-first folder structure, Riverpod State Notifier, Freezed immutable data models, GoRouter setup.
- **Unity ([unity.md](../../prompts/unity.md))**: SOLID-compliant scripts, ScriptableObject design, memory optimization via Object Pooling, Zenject Dependency Injection.
- **SQL Server ([sql.md](../../prompts/sql.md))**: Slow query optimization, smart index design, safe SQL migrations, execution plan analysis.
- **Docker & DevOps ([docker.md](../../prompts/docker.md))**: Lightweight multi-stage Dockerfiles, production docker-compose, container health checks.
- **Clean Code ([clean-code.md](../../prompts/clean-code.md))**: Clean code principles — naming, function structure, error handling, effective comments.
- **Code Review ([review.md](../../prompts/review.md))**: Comprehensive review checklist — security, performance, architecture, maintainability.
- **Testing ([testing.md](../../prompts/testing.md))**: General testing strategies — unit tests, integration tests, end-to-end, mock/stub patterns.

## 🛠️ How to Use Prompts Effectively
1. Identify the language/framework you're working with.
2. Open the corresponding file in the `prompts/` directory.
3. Fill in the information in square brackets `[...]` with your actual project context.
4. Send the completed prompt to Codex CLI for best results.
